## Overview The decision support logic embedded within the Core Information Standard ensures data quality, clinical safety, and appropriate care coordination. These decision tables define automated rules that trigger alerts, validations, and workflow guidance based on documented information. ## DT001: Reasonable Adjustments Alert Generation ### Description Automatically identifies when reasonable adjustments should be flagged based on documented conditions, communication needs, or explicit requests. Ensures compliance with Equality Act 2010 requirements. ### Input Elements - Diagnosis/Problem list entries - Communication preferences - Sensory impairments documented - Learning disability flag - Mobility requirements ### Decision Table | Condition ID | Input Conditions | Output Action | Annotation | |---|---|---|---| | DT001.1 | Diagnosis contains "learning disability" OR "intellectual disability" | Set Adjustment Flag = TRUE, Prompt for specific needs | Legal requirement to identify | | DT001.2 | Diagnosis contains "dementia" OR "cognitive impairment" | Set Adjustment Flag = TRUE, Add "longer appointments" | Cognitive support needed | | DT001.3 | Preferred language ≠ English | Set Communication Need = "Interpreter required" | Ensure language specified | | DT001.4 | Diagnosis contains "deaf" OR "hearing impairment" | Set Communication Need = "BSL interpreter" or "Hearing loop" | Check preference | | DT001.5 | Diagnosis contains "blind" OR "visual impairment" | Set Information Format = "Large print" or "Audio" | Confirm format preference | | DT001.6 | Mobility aid documented OR wheelchair user | Set Access Need = "Step-free access required" | Physical accessibility | | DT001.7 | Autism spectrum condition documented | Set Adjustment Flag = TRUE, Add "Quiet environment" | Sensory considerations | ## DT002: Safeguarding Risk Escalation ### Description Evaluates combinations of risk factors to determine when safeguarding alerts should be raised and which professionals should be notified. ### Input Elements -
