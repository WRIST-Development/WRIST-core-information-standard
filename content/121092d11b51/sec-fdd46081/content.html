## Decision-Support Logic The Core Information Standard v2.0 incorporates decision support to guide appropriate documentation and care delivery based on person characteristics and circumstances. ### Decision Table: Reasonable Adjustment Requirements \*\*Title\*\*: Determining Required Reasonable Adjustments \*\*Description\*\*: Based on identified impairments and communication needs, determine which reasonable adjustments must be provided under the Equality Act 2010. | Condition (Input) | Output Action | | --- | --- | | Impairment = Sensory (sight) | Provide information in alternative formats (large print, audio, Braille) | | Impairment = Sensory (hearing) | Provide BSL interpreter or written communication | | Impairment = Learning disability | Provide easy-read materials; allow extra time; support person present | | Impairment = Dementia | Simplify communication; involve carer; provide visual cues | | Impairment = Physical disability | Ensure accessible facilities; adjust appointment times; home visits | | Communication need = Non-English speaker | Provide professional interpreter (not family member) | | Multiple impairments present | Combine adjustments; prioritize based on severity | ### Decision Table: Mental Capacity Assessment Triggers \*\*Title\*\*: When to Conduct Mental Capacity Assessment \*\*Description\*\*: Triggers that indicate a mental capacity assessment should be conducted for specific decisions. | Condition (Input) | Output Action | | --- | --- | | Major treatment decision required | Assess capacity for specific treatment decision | | Person has dementia diagnosis | Assess capacity for each significant decision | | Person has learning disability | Consider capacity; adjust communication method | | Refusal of recommended treatment | Assess capacity to refuse treatment | | Significant change in mental state | Reassess capacity for ongoing decisions | | Financial or legal decision needed | Assess capacity for specific decision type | | Conflicting views (person vs family) | Formal capacity assessment required | ### Decision Table: Safeguarding Response Requirements \*\*Title\*\*: Safeguarding Response Based on Risk Type \*\*Description\*\*: Determines required actions based on identified safeguarding concerns. | Condition (Input) | Output Action | | --- | --- | | Risk of self-harm identified | Immediate risk assessment;
